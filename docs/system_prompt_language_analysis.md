## 5. 핵심 발견

### 5.1 ZH 시스템 프롬프트의 기술 항목 하락

ZH 시스템 프롬프트 사용 시 **의상·카메라 기술 항목이 일관적으로 하락**:

| 항목 | EN sys 평균 | ZH sys 평균 | 하락폭 |
|------|:---:|:---:|:---:|
| 네크라인 | 7.3 | 5.5 | -1.8 |
| 소재/질감 | 6.6 | 4.8 | -1.9 |
| 노출부위 | 6.4 | 5.6 | -0.8 |
| DoF | 5.8 | 5.3 | -0.5 |

원인: EN 기술 용어(off-shoulder, scoop neckline, shallow depth of field)가 모델의 학습 분포에서 더 **정확한 어텐션 앵커** 역할. ZH 시스템 프롬프트는 이 앵커가 약해져 일반적/미학적 묘사로 치환됨.

### 5.2 ZH 시스템 프롬프트의 미학 항목 상승

| 항목 | EN sys 평균 | ZH sys 평균 | 상승폭 |
|------|:---:|:---:|:---:|
| 색상팔레트 | 7.3 | 8.8 | +1.5 |
| 감정 | 5.8 | 7.3 | +1.5 |
| 카메라앵글 | 4.8 | 5.1 | +0.3 |

ZH 시스템 프롬프트가 모델을 "미학 모드"로 유도 → 색상, 분위기, 감정 어휘 풍부해지지만 기술적 정밀도 상실.

### 5.3 체태곡선·중심분포: 출력 언어가 지배적

| 조합 | 체태곡선 | 중심분포 |

- 결론: **출력 언어가 시스템 프롬프트 언어보다 지배적**

## 6. 원인 분석: 왜 ZH 시스템 프롬프트가 더 나쁜가?

### 6.1 "이해 못함"이 아니라 "활성화 패턴 차이"

Qwen3.5-9B는 ZH 시스템 프롬프트를 **충분히 이해**함:
- 형식 준수율 EN sys 9.4/10 vs ZH sys 9.8/10 (동등)
- 주체 첫문장 모두 10/10
- 포즈 방향, 배경 묘사 등 핵심 요소 정확히 반영

문제는 **"이해"가 아니라 "정밀도 활성화"**:

```
EN system prompt: "neckline type (e.g., sweetheart, scoop, V-neck)"
→ 모델이 "off-shoulder", "square neckline" 등 정확한 레이블 출력

ZH system prompt: "领口类型（如心形领、圆领、V领）"
→ 모델이 "领口较低", "露肩设计" 등 서술적 묘사로 대체
```

### 6.2 이중 언어 처리 메커니즘

```
[EN sys → ZH out] (enzh, 최적)
├── EN 시스템 프롬프트 처리 → 기술 체크리스트 정밀 활성화
├── ZH 출력 생성 → 미학 어휘 활용 (曲线, 线条, 体态)
└── 결과: 기술 정밀도 + 미학 풍부함 = 최적 조합

[ZH sys → ZH out] (zhzh, 차선)
├── ZH 시스템 프롬프트 처리 → 미학 모드 과도 활성화
├── ZH 출력 생성 → 이미 미학 모드이므로 추가 이점 없음
└── 결과: 미학 과잉 + 기술 부족 = 불균형

[EN sys → EN out] (enen)
├── EN 시스템 프롬프트 처리 → 기술 정밀
├── EN 출력 생성 → 미학 어휘 부족
└── 결과: 기술 정밀 + 미학 부족

[ZH sys → EN out] (zhen)
├── ZH 시스템 프롬프트 처리 → 기술 앵커 약화
├── EN 출력 생성 → 기술 어휘 일부 복원
└── 결과: EN sys → EN out과 실질 동등
```

### 6.3 핵심 통찰

> **EN 시스템 프롬프트는 "기술 앵커" 역할을 하고, ZH 출력 언어는 "미학 증폭기" 역할을 한다.**
> 두 역할을 분리해서 각각 최적 언어를 배정하는 enzh 조합이 최적.
> 같은 언어로 통일하면 한쪽이 과잉되고 다른 한쪽이 부족해진다.
